ML on Quantum Simulators

Objective: The main goal of this project was to simulate machine learning on a quantum environment and explore the booming field of Quantum Computers

Approach: This was a research based project on Quantum Computers and it’s simulators under ’Dr. Satyadhyan Chickerur’, a renowned professor at School of Computer Science and Engineering, KLE Tech. Through this project, I had an opportunity to learn the basics of quantum computers and its working. The different properties of the quantum particles such as quantum entanglement, superpositions were studied as a part of pre-requisites for the research. Through the course of project, we were able to gather continuous insights and implement them.

Takeaways: I was able to explore a relatively fresh and booming technology of Quantum Computers with concepts like Quantum Entaglement, Superposition, Quantum Logic gates, Quantum Circuits. Machine Learning frameworks like TFQ (Tensorflow Quantum), and cirq, seaborn, keras etc along with some key ML concepts like CNN.

Result: I was able to complete the project by successfully implementing Machine Learning for binary classification of the images. I was able to implement CNN on a quantum simulated circuit. For comparison purpose, a traditional ML was implemented. The quantum version bore an accuracy of 69 percent initially which was later raised to 73 percent after some tweaks. All the project files including report is uploaded on github.

CFD

Objective: To implement Computational Fluid Dynamics simulation using High Performance Computers on a 2D silhouette image.

Approach: This project was done as a 5 member team including myself. We as a team first happened to do our literature survey on the basics of Computational Fluid Dynamics. This subject is the study of fluid particles movements. Before we could program something, it was crucial for us to know the fluid particle behaviours with required equations. We happened to use a framework by name ’mechsys’ to help simulate fluid particles with the attributes that can be tweaked as desired (ex- viscocity, fluid molecule weight etc). This was later fed with a series of 9 equations known as ’Navier-Stokes’ equation.

Takeaways: A good knowledge on the fluid flow behaviour on a steady state solid, lattice boltzmann method, navier-stokes equations, Tensorflow, opencv and mechsys.

Result: We were able to generate the output image with the fluid flow simulation along with the pressure on the particles being indicated with color coding on the particle themselves.

Software Development Intern at INVENTO ROBOTICS (Mar' 2021 - Jul' 2021)

Objective: To develop a fleet management Dashboard for connecting with the robots

Approach: The robots are filled with a lot of features as desired by the clients. It became difficult for them to keep track of everything the robot does. So, we had to build a solution where the robot is tracked and maintained. A web portal was a very fine choice to do so as it has the potential to be highly scalable. Earlier, a lot all our users found it difficult to get familiar with all the softwares, hardwares, operating operations, and manuals. Now we are building our own custom User Dashboard for all these operations. The main stack used was django and ROS. The communication between the server and ROS master is facilitated by ROS bridge which mimics all the topics and nodes from ROS to main server. The werbrtc channel uses ngrok tunneling for the camera feed and the sensors uses the socket communication.

Takeaways : Through this project, I was able to get experience in a lot of the front-end technologies and frameworks like bootstrap, momentjs, easypiejs, createjs, jQuery etc, backend frameworks includes django to build some of the required APIs. Other technolgies and concepts learnt are ROSLIB, ROS2D, rosbridge, webrtc, socket connections, APIs etc. Along with technologies, I was also provided with an opportunity to do the technical documentation of the front-end stack using Latex.

Result : All ROS related heavy softwares like gazebo and rviz are replaced with our own Dashboard. We are getting all the features like the robot map, sensor data and camera feed live on the dashboard which is now being used by the clients.
